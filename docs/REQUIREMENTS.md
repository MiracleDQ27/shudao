# 需求文档

## 核心需求

一个读书修行游戏化系统，把"读书 → 写理解 → 成长"变成可视化的打怪升级体验。

## 功能需求

### 系统侧
| 需求 | 说明 | 状态 |
|------|------|------|
| 微信读书数据接入 | 拉取书架、阅读进度、时长、连续天数 | 已对接 ✅ |
| IMA 理解记录 | IMA DeepSeek 陪读后自动追加结构化笔记 | 指令已定稿 ✅ |
| Claude 结算引擎 | 拉双源数据 → 交叉验证 → 计算 XP/维度/Boss → 写 data.json | 设计中 |
| data.json 数据模型 | 等级、七维、Boss、修习、知识点等字段 | v2 已重构 ✅ |

### 网页可视化侧（待设计）

| 模块 | 说明 | 状态 |
|------|------|------|
| 玩家面板 | 等级、称号、XP 进度条、修习天数 | 待设计 |
| 七维雷达图 | 七道能力分布，段位展示 | 待设计 |
| Boss 链/战历 | 最近击败的 Boss 记录 | 待设计 |
| 书架 | 在读/已完成/计划 | 待设计 |
| 修习统计 | 连续天数、总天数、趋势 | 待设计 |

## 非功能需求

- 单人使用，不需要账号系统
- 纯前端可视化，GitHub Pages 托管
- 数据来源为 data.json，Claude 负责写入
- 响应式，手机可查看（主要桌面端）
- 视觉风格：读书 + 修仙融合
