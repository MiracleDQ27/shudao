# 需求文档

> 书道系统的需求、思路和迭代记录

## 核心理念

把读书变成打怪升级，核心不是"让你多读"，而是**帮你验证读懂了没有**。

这是控制论的反馈闭环：输出 → 检测 → 调整。游戏化是装饰层，不是替代层。

## 用户故事

- 作为读者，我想在手机端随时记录读书进度和输出理解
- 作为读者，我想获得即时反馈——哪些理解对了、哪些有偏差
- 作为读者，我想看到自己的成长可视化（等级、维度、Boss进度）
- 作为读者，我想在电脑上打开一个好看的页面展示我的修行成果
