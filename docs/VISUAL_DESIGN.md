# 书道 · 前端视觉设计

> 洞府修士面板 — 个人修界面板可视化设计
> 定稿日期：2026-05-22

---

## 一、设计定位

**场景**：独处洞府时打开的个人修界面板  
**氛围关键词**：静 · 雅 · 仙气 · 质感  
**风格**：暗色调国风 · 立体感 · 仙气飘飘  

背景有若隐若现的远山轮廓和流动的云雾，不抢眼但让背景有呼吸感。

### 模块划分

| 区域 | 模块 | 内容 |
|------|------|------|
| 顶部通栏 | 状态栏 | 境界 · 等级 · 称号 · 书道标题 · 修习天数 · 连续天数 · 加成倍率 |

## 四、核心交互

### 4.1 灵根雷达图
- 七边形 SVG 雷达图，每边代表一个维度
- 维度数据（点数）决定顶点距离中心的远度
- 点击左侧维度行 → 雷达图对应维度高亮
- 空数据时显示中心极小占位多边形

### 4.2 书域山河图
- 2D 风格化地图，不用 Three.js
- 七道领域用圆形渐变标注区域
- 书籍标记为发光点（已读/在读/待读不同透明度）
- 玩家位置用一个发光圆环表示
- 图例：已探索书籍 / 待探索 / 当前位置

### 4.3 整体交互
- hover/click 有微反馈（非本阶段实现细节）
- 数据为空时展示引导文字

## 七、技术约束

- 纯前端 HTML/CSS/JS，无框架
- 数据源：data.json（fetch 加载）
- 地图和雷达图用 Canvas/SVG，不用 Three.js（性能考虑）
- GitHub Pages 托管
- 手机端后续适配
